# 记忆系统说明

## 分层记忆目录

本目录用于存储领域层（软件工程师）的知识和经验。

---

## 记忆内容

### 适合记录的内容

1. **技术决策**
   - 架构选择及原因
   - 技术栈评估结果
   - 设计模式应用经验

2. **问题解决方案**
   - 常见问题的解决方法
   - 调试技巧
   - 性能优化经验

3. **最佳实践**
   - 代码规范总结
   - 工作流优化
   - 工具使用技巧

4. **经验教训**
   - 踩过的坑
   - 避免的错误
   - 改进建议

---

## 记忆组织

建议按主题创建独立文件：
- `architecture-decisions.md` - 架构决策
- `debugging-tips.md` - 调试技巧
- `performance-optimization.md` - 性能优化
- `common-issues.md` - 常见问题

---

## 与全局记忆的区别

- **全局记忆**：跨领域的通用知识
- **领域记忆**：软件工程特定的知识
- **应用记忆**：特定工作场景的经验
- **项目记忆**：具体项目的知识

---

## 使用建议

- 定期整理和更新
- 保持内容简洁
- 使用清晰的标题和分类
