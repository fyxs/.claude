## 两种初始化方式的区别

### 1. 常规 .claude 目录初始化

**触发关键词：**
- "初始化 .claude"
- "创建 .claude 目录"
- "设置 .claude 结构"

**操作内容：**
创建标准的 `.claude` 目录结构：
```
.claude/
├── rules/
│   └── common/
│       └── workspace.md
└── settings/
```

**执行方式：**
```bash
mkdir -p "目标路径/.claude/rules/common"
mkdir -p "目标路径/.claude/settings"
# 创建基本的 workspace.md
```

---

### 2. 项目级智能初始化（init-project skill）

**触发关键词：**
- "初始化项目"
- "智能初始化"
- "基于项目初始化"
- 明确提到使用 init-project skill

**操作内容：**
- 分析项目类型和技术栈
- 智能生成项目特定的规则
- 创建详细的项目配置
- 可能包含 steering、agents 等高级配置

**执行方式：**
使用 init-project skill

---

## 判断标准

| 场景 | 使用方式 |
|------|---------|
| 用户只说"初始化 .claude" | 常规目录初始化 |
| 用户说"初始化项目的 .claude" | 常规目录初始化 |
| 用户说"智能初始化项目" | init-project skill |
| 用户明确提到 init-project | init-project skill |

**默认原则：** 除非用户明确要求智能初始化或使用 init-project，否则使用常规目录初始化。

---

## 常规初始化详细流程

### 步骤 1：创建目录结构

```bash
# 创建基础目录
mkdir -p "目标路径/.claude/rules/common"
mkdir -p "目标路径/.claude/settings"
mkdir -p "目标路径/.claude/memory"
mkdir -p "目标路径/.claude/templates"
```

### 步骤 2：创建基础配置文件

**创建 architecture.md**（如果是领域层或应用层）：
- 说明当前层级的职责
- 列出规则文件索引
- 定义配置优先级

**创建 workspace.md**（已废弃，内容合并到 architecture.md）：
- 工作区信息
- 目录结构说明
- 通用规则

### 步骤 3：根据层级添加特定配置

**全局层初始化**：
- 创建通用规则文件
- 配置全局 MCP 服务器
- 设置全局记忆

**领域层初始化**：
- 创建领域特定规则
- 配置领域 MCP 服务器
- 设置领域记忆

**应用层初始化**：
- 创建应用场景规则
- 配置应用特定工具
- 设置应用记忆

**项目层初始化**：
- 创建项目配置
- 配置项目 MCP
- 设置项目文档

## 智能初始化详细说明

### init-project skill 功能

**自动分析**：
- 检测项目类型（前端/后端/全栈）
- 识别技术栈（Vue/React/Node.js 等）
- 分析项目结构

**智能生成**：
- 项目特定的规则文件
- 技术栈相关的配置
- 开发工作流规则
- 测试和构建规则

**高级配置**：
- steering 规则（条件包含）
- 自定义 agents
- 项目模板
- 钩子配置

### 何时使用智能初始化

**适用场景**：
- 新项目启动，需要完整配置
- 复杂项目，需要定制化规则
- 团队项目，需要标准化配置
- 需要快速搭建开发环境

**不适用场景**：
- 只需要基础目录结构
- 临时测试项目
- 已有配置，只需微调

## 错误处理与故障排查

### 常见问题 1：目录已存在

**问题**：`.claude` 目录已存在

**解决方案**：
- 检查现有配置
- 确认是否需要覆盖
- 备份现有配置后再初始化

### 常见问题 2：权限不足

**问题**：无法创建目录或文件

**解决方案**：
- 检查目录权限
- 使用管理员权限
- 更换目标路径

### 常见问题 3：路径错误

**问题**：目标路径不存在或无效

**解决方案**：
- 验证路径格式
- 确认父目录存在
- 使用绝对路径

## 最佳实践

### 1. 分层初始化

**原则**：从上到下逐层初始化

**顺序**：
1. 全局层（如需要）
2. 领域层
3. 应用层
4. 项目层

### 2. 渐进式配置

**原则**：先创建基础结构，再逐步完善

**步骤**：
1. 创建目录结构
2. 添加基础配置
3. 根据需求添加规则
4. 持续优化和调整

### 3. 文档先行

**原则**：先创建 architecture.md，说明层级职责

**内容**：
- 层级定位
- 职责范围
- 规则索引
- 快速参考

### 4. 避免过度配置

**原则**：只添加必要的配置

**建议**：
- 不要复制所有上层规则
- 只覆盖需要修改的配置
- 保持配置简洁
