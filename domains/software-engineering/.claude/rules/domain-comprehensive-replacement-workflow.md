# 全面替换工作流规则

> 规则 ID: DOMAIN-WORKFLOW-001
> 版本: 1.0.0
> 创建日期: 2026-03-09
> 最后更新: 2026-03-09
> 状态: 发布

> 本规则定义了"全面替换"类任务的标准工作流，避免遗漏和过早完成。

## 适用场景

当任务包含以下关键词时，必须遵循本规则：
- "全面替换"、"全部替换"
- "所有的 XXX"、"项目中的 XXX"
- "批量替换"、"统一替换"

**示例任务：**
- "全面将 TeJobV2，TeLabelV2，TeTitleV2 组件进行替换"
- "替换项目中所有的 console.log"
- "统一更新所有 API 调用"

---

## 范围识别（CRITICAL）

**明确搜索范围：**

1. **有明确范围限定**
   - "全面替换 xxx 模块下的..." → 只在 xxx 模块目录搜索
   - "替换 src/components 下的..." → 只在该目录搜索
   - 示例：`Grep pattern="..." path="指定目录"`

2. **无范围限定或全局范围**
   - "全面替换项目中的..." → 整个项目搜索
   - "替换所有的..." → 整个项目搜索
   - 示例：`Grep pattern="..." path="项目根目录"`

**判断原则：**
- 用户明确指定目录/模块 → 限定范围搜索
- 用户说"项目中"、"所有" → 全局搜索
- 不确定时 → 询问用户确认范围

---

## 标准工作流（MANDATORY）

### 第一步：全面搜索

**目的：** 列出所有需要修改的文件

**操作：**
```bash
# 使用 Grep 搜索所有匹配的文件
Grep pattern="目标模式" output_mode="files_with_matches"
```

**输出：**
- 明确列出所有需要修改的文件数量
- 记录文件列表

**示例：**
```
找到 5 个文件需要替换：
1. file1.vue
2. file2.vue
3. file3.vue
4. file4.vue
5. file5.vue
```

---

### 第二步：逐个替换

**目的：** 系统化地完成所有替换

**操作：**
- 使用 TodoWrite 创建任务列表
- 逐个文件进行替换
- 每完成一个文件，立即标记为 completed

**原则：**
- 不要跳过任何文件
- 不要批量处理（容易出错）
- 保持任务列表实时更新

---

### 第三步：再次搜索验证（CRITICAL）

**目的：** 确保没有遗漏

**操作：**
```bash
# 使用相同的搜索模式再次搜索
Grep pattern="目标模式" output_mode="files_with_matches"
```

**判断标准：**
- ✅ 搜索结果为 0 → 替换完成
- ❌ 搜索结果 > 0 → 有遗漏，继续替换

**CRITICAL：** 这一步是必需的，不能跳过！

---

### 第四步：运行检查

**目的：** 验证代码质量

**操作：**
```bash
# TypeScript 类型检查
npx tsc --noEmit

# ESLint 代码检查
npm run lint
```

**注意：**
- 这些检查只能验证语法正确性
- 不能替代第三步的搜索验证

---

## 常见错误

### 错误 1：过早完成

**问题：**
- 替换了部分文件后，直接运行检查
- 没有验证是否还有遗漏

**后果：**
- 项目中仍有未替换的文件
- 用户需要重新指出

**正确做法：**
- 必须执行第三步：再次搜索验证

---

### 错误 2：依赖工具检查

**问题：**
- 认为 TypeScript/ESLint 通过就是完成
- 没有搜索验证

**后果：**
- 工具检查只能验证语法
- 无法发现未替换的文件

**正确做法：**
- 工具检查是第四步，不是第三步
- 必须先搜索验证，再运行工具检查

---

### 错误 3：理解偏差

**问题：**
- 把"全面替换"理解成"替换已知的文件"
- 没有全面搜索

**后果：**
- 只替换了部分文件
- 遗漏了其他文件

**正确做法：**
- "全面"意味着"搜索所有 + 替换所有"
- 第一步必须是全面搜索

---

## 检查清单

**在声称任务完成前，必须确认：**

- [ ] 已执行全面搜索，列出所有文件
- [ ] 已逐个替换所有文件
- [ ] 已再次搜索验证，结果为 0
- [ ] 已运行 TypeScript 类型检查
- [ ] 已运行 ESLint 代码检查

**只有所有检查项都通过，才能声称任务完成。**

## 工作流图示

```
用户请求"全面替换 XXX"
    ↓
第一步：全面搜索
    ↓ 找到 N 个文件
第二步：逐个替换（使用 TodoWrite 跟踪）
    ↓ 替换完成
第三步：再次搜索验证
    ↓ 结果为 0？
    ├─ 是 → 第四步：运行检查 → 完成
    └─ 否 → 返回第二步，继续替换
```

## 实践案例

### 案例：替换 v1 组件为 v2 组件

**任务：** "全面将 TeJobV2，TeLabelV2，TeTitleV2 组件进行替换"

**执行流程：**

1. **全面搜索**
   ```bash
   Grep pattern="\bTeLabel\b|\bTeJob\b|\bTeTitle\b"
   # 结果：找到 5 个文件
   ```

2. **逐个替换**
   - 创建 TodoWrite 任务列表（5个任务）
   - 逐个替换，每完成一个标记为 completed

3. **再次搜索验证**
   ```bash
   Grep pattern="\bTeLabel\b|\bTeJob\b|\bTeTitle\b"
   # 结果：No files found ✅
   ```

4. **运行检查**
   ```bash
   npx tsc --noEmit  # 通过 ✅
   npm run lint      # 通过 ✅
   ```

5. **声称完成**
   - 所有检查项通过
   - 可以声称任务完成

---

## 总结

**核心原则：**
1. 全面搜索 → 列出所有
2. 逐个替换 → 不遗漏
3. 再次验证 → 确保完成
4. 运行检查 → 保证质量

**记住：** 第三步（再次搜索验证）是关键，不能跳过！

1. **第一步：全面搜索**

   - 识别跳过了第三步（再次搜索验证）

5. **执行第三步验证**
   - 再次搜索 `oldAPI`
   - 发现还有 2 个文件未替换（结果 > 0）
6. **继续替换**
   - 返回第二步，替换剩余的 2 个文件
   - 再次验证，结果为 0
   - 才能声称完成
