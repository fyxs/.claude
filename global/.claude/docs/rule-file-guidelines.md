# 规则文件创建指南

> 确保所有规则文件达到缓存阈值，提高效率

## 核心要求

### 最小文件大小（CRITICAL）

**缓存阈值：≥1024 tokens（约 768 words）**

- ❌ 小于 1024 tokens：不会被缓存，每次会话重新处理
- ✅ 1024-1500 tokens：可缓存，但接近边缘
- ✅ 1500-3000 tokens：理想范围
- ⚠️ 大于 3000 tokens：考虑拆分

## 创建规则文件前检查

**必须回答的问题：**

1. 这个规则文件是否会达到 1024+ tokens？
2. 如果太小，能否合并到现有文件？
3. 如果必须独立，如何扩充内容？

## 验证方法

**创建文件后立即运行：**

```bash
# 检查单个文件
wc -w ~/.claude/rules/common/your-file.md

# 检查所有规则文件
~/.claude/scripts/check-cache-efficiency.sh
```

**判断标准：**
- < 768 words：必须合并或扩充
- 768-1125 words：建议扩充
- 1125-2250 words：理想范围
- > 2250 words：考虑拆分

## 最佳实践

1. **优先合并** - 将相关的小规则合并到一个文件
2. **主题分组** - 按功能/主题组织，而非按类型
3. **定期审查** - 每月检查规则文件大小
4. **自动检查** - 使用脚本验证

## 自动检查

**添加到工作流：**

```bash
# 每次修改规则文件后运行
~/.claude/scripts/check-cache-efficiency.sh
```

**预期输出：**
- ✅ 所有文件都达到缓存阈值
- ⚠️ 发现 X 个文件低于阈值

---

**记住：** 规则文件太小会导致缓存失效，增加 60%+ 的 token 成本。
