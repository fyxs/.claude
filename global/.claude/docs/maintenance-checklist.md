# 规则文件维护清单

> 定期审查，保持高效

## 每周检查（推荐）

**执行：**
```bash
~/.claude/scripts/check-cache-efficiency.sh
```

**检查项：**
- [ ] 所有规则文件 ≥ 1024 tokens
- [ ] 无重复内容
- [ ] 无过期规则

## 每月审查（必需）

**检查项：**
- [ ] 规则文件大小分布合理
- [ ] 缓存命中率 > 80%
- [ ] 文件数量控制在 5-10 个
- [ ] 清理过期的计划文件和会话记忆

## 触发审查的信号

**立即审查如果：**
- 创建了新的规则文件
- 拆分了现有规则文件
- Token 成本异常增加
- 响应速度变慢

## 快速修复

**文件太小（< 1024 tokens）：**
1. 合并到相关文件
2. 扩充内容
3. 删除并整合到其他文件

**文件太大（> 3000 tokens）：**
1. 按主题拆分
2. 确保每个拆分后的文件 ≥ 1500 tokens
